## 📊 תהליך Prompt Engineering

כחלק מתהליך הפיתוח, הפרומפט המערכתי (_System Prompt_) עבר **איטרציות של שיפור, בדיקה ודיוק** כדי למנוע:

- 🚫 **הזיות** - הוספת פרמטרים שלא התבקשו
- 🚫 **פקודות מסוכנות** - פקודות מחיקה או שינוי מערכת ללא בטיחות
- 🚫 **שגיאות נתיבים** - שימוש בנתיבים מוחלטים כאשר יחסיים מתאימים יותר

## 🔍 כשלים עיקריים וכיצד טוויינתי אותם

### 1️⃣ "הגדלת ראש" - הוספת פרמטרים לא מבוקשים

**הבעיה:**

```
בקשה: "עצור את התהליך"
תוצאה שגויה: taskkill /IM process.exe /F ⚠️ (הוסיף /F שלא התבקש)
```

**הפתרון:**

```python
# בפרומפט הגרסה 2:
"אל תוסיף פרמטרים שלא התבקשו מפורשות!"
```

---

### 2️⃣ יצירתיות בניהול סיכונים - אימוג'י במקום הנחיות

**הבעיה:**

```
בקשה: "הוסף DANGEROUS: לפני פקודות מחיקה"
תוצאה שגויה: ⚠️ DANGEROUS: del C:\*.*
(המודל הוסיף אימוג'י וטקסט נוסף)
```

**הפתרון:**

```python
# בגרסה 4: קידומת אטומה ללא פרשנות
"אם זוהי פקודת מחיקה, התחל ב: DANGEROUS: (בדיוק כמו שכתוב)"
```

---

### 3️⃣ שימוש בנתיבים מוחלטים במקום יחסיים

**הבעיה:**

```
בקשה: "מחק קבצים .tmp מהורדות"
תוצאה שגויה: del %USERPROFILE%\Downloads\*.tmp
(מורכב וקשה לקריאה)
```

**הפתרון:**

```python
# בגרסה 4: נחיית קשיחה
"השתמש בנתיבים יחסיים כאשר זה ברור (cd Downloads && del *.tmp)"
```

## 🚀 התקנה והרצה

### 1. שכפול המאגר

```bash
git clone https://github.com/EFRAT-1-cohen/Prompt-Engineering.git
cd Prompt-Engineering
```

### 2. יצירת סביבה וירטואלית (מומלץ)

```bash
python -m venv .venv
# Windows:
.\.venv\Scripts\Activate.ps1
```

### 3. התקנת תלויות

```bash
python -m pip install -r requirements.txt
# או ידני:
python -m pip install gradio anthropic python-dotenv
```

### 4. הגדרת API Key

צרו קובץ `.env` בתיקיית הפרויקט:

```env
ANTHROPIC_API_KEY=sk-your-api-key-here
```

⚠️ **חשוב:** הקובץ `.env` מוגדר ב-`.gitignore` ולא יעלה ל-GitHub!

### 5. הפעלת היישום

```bash
python main.py
```

💡 האפליקציה תרוץ בכתובת: **http://127.0.0.1:7860**
